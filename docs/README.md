# 문서 규칙

이 repo의 `docs/`를 다룰 때 따르는 규칙.

## 규칙

- 파일명은 영문 kebab-case (`api-spec.md`).
- 문서를 수정하면 개발 단위로 커밋한다.
- 문서는 두 종류로 나눠 다룬다.
  - **레퍼런스** (현재 상태만): 제자리에서 수정하고 낡은 내용은 지운다. 날짜별 사본을 만들지 않는다 (이력은 git history).
  - **기록(인수인계)**: `dev-log.md` — "현재 상태·최근 결정·다음 할 일"을 담는 살아있는 문서. 최신을 맨 위에 쓰고, 오래되거나 끝난 항목은 자유롭게 압축·정리한다. 변경의 정확한 이력은 git이 보관하므로 완전한 연대기로 유지하려 애쓰지 않는다.
- 새 문서를 추가하면 아래 표에 한 줄 등록한다.

## 문서 목록

| 파일 | 내용 | 종류 |
|---|---|---|
| `plan.md` | 구현 플랜 | 레퍼런스 |
| `api-spec.md` | API 요청/응답 명세 | 레퍼런스 |
| `dev-log.md` | 진행상황·결정 기록 | 기록 |
