### 1. 프로젝트 소개

#### 1.1. 개발배경 및 필요성

**[거시환경 분석]**

- **부산 도심 상가 공실이 지속적으로 심화되고 있음**
  - 2024년 2분기 기준 부산대 앞 상권 공실률은 23.4%(전국 5위) 수준이며, 금정구 내 341개 점포 중 95곳이 공실 상태
  - 부산 중대형 상가 공실률은 15.3%로, 서울(9.3%) 대비 약 1.6배 수준 (출처: 한국부동산원, 상업용부동산 임대동향조사, 2026 1분기)

- **기존 도시재생 정책은 자금 흐름 구조 부재로 실효성이 제한됨**
  - 도시재생 뉴딜사업 거점시설 80개 중 약 20%(15개)가 운영 주체 및 활용 용도를 정하지 못한 채 방치
  - 수익 모델과 자금 순환 구조 부재가 주요 원인 (출처: 뉴스타파·단비뉴스, '도시재생 10년, 길을 잃었나', 2023)

- **실내농업·스마트팜 시장은 빠르게 성장하고 있음**
  - 국내 스마트팜 시장은 2020년 2.4억 달러에서 2025년 4.9억 달러 규모로 성장, 연평균 성장률 약 15.5% (출처: MarketsandMarkets, 농림축산식품부 재인용)
  - 부산시 도시농업 참여 가구는 약 26.9만 가구로, 전체의 17.2% 수준 (출처: 부산광역시, 2023)

**[산업환경 분석]**

- **도심 공실과 스마트팜 시장 간 연결 구조가 부재함**
  - 공실 상가 미니팜 전환 비용은 공간당 3,000만~5,000만 원 수준으로, 개인 단독 부담이 어려움
  - 국내 시설원예농가의 스마트팜 도입률은 1.5%에 그치며, 도입 장애 요인 중 초기 투자 비용 부담이 가장 높은 비중 (출처: 한국무역협회, 2024)

**[경쟁환경 분석]**

- **2026년 STO 제도화로 소액 자금 조달 경로가 마련됨**
  - 2026년 1월 전자증권법·자본시장법 개정안이 국회를 통과해 분산원장 기반 토큰증권 발행 및 투자계약증권 유통이 허용됨 (시행 2027년 1월 예정)
  - 2030년 국내 STO 시장 규모는 약 30–60조 원, 2026–2027년이 시장 진입 적기 (출처: 자본시장연구원 등, 2024~2025)

- **기존 STO 플랫폼은 자금 조달 경로만 열었을 뿐, 자금 통제 메커니즘이 부족함**
  - 카사·루센트블록 등 기존 STO 플랫폼은 공모 후 운영자에게 자금을 일괄 지급하는 구조
  - 카사 DABS 등 일부 자산이 공모가 대비 10% 이상 가치 하락 사례 발생, 투자자 보호 부재 비판 지속

#### 1.2. 개발 목표 및 주요 내용

본 프로젝트는 실제 토큰증권 발행이 불가하므로, **자금 통제 메커니즘의 작동 가능성을 코드 수준에서 검증**하는 것을 목표로 합니다.

- 결제 입출구(원화 입출금·KYC·공시)는 모의 처리하되, 핵심 메커니즘 레이어(토큰 발행·에스크로 락업·마일스톤 자동 집행)는 **Polygon Amoy 테스트넷에 실제 배포**하여 작동을 검증합니다.

**검증 명제 3가지:**

| 검증 명제 | 대응 조건 | 데모 시연 방식 |
|:---|:---|:---|
| ① 마일스톤 미달 시 자금이 풀리지 않음을 시연 가능하다 | 시간 축 위험 분산 | 실패 케이스: 영수증 미달 → 컨트랙트 자동 차단 |
| ② 사람의 임의 개입으로 자금 풀림을 강제할 수 없음을 시연 가능하다 | 운영자 재량 차단 | 관리자 강제 호출 시도 → 컨트랙트 거부 확인 |
| ③ 마일스톤 통과 시 자금 풀림이 추가 승인 없이 자동 집행됨을 시연 가능하다 | 집행의 무조건성 | 다중 신호 통과 → 컨트랙트 자동 호출 → 트랜잭션 해시 발급 |

#### 1.3. 세부내용

서비스의 사용자는 일반 투자자, 청년 창업자, 건물주, 플랫폼 관리자이며, 각각의 니즈에서 도출한 핵심 기능은 아래와 같습니다.

| 사용자 | 니즈 | 핵심 기능 |
|:---|:---|:---|
| 일반 투자자 | 미니팜 투자 프로젝트를 한 곳에서 확인하고 소액으로 간편하게 참여하고 싶다. 내 돈이 투명하게 사용되고, 약속 미이행 시 자금이 자동 보호되길 원한다. | 프로젝트 탐색 대시보드, 라이브 대시보드(에스크로 잔액·마일스톤·IoT), 소액 청약 플로우(토큰 발행 → 에스크로 락업) |
| 청년 창업자 | 대규모 자본 없이 사업을 시작하고 싶고, 마일스톤 완료 시 자금이 자동 집행되길 원한다. | STO 공모 등록 및 청약 모집, 마일스톤 검증 자료 제출(영수증·사진), 마일스톤 진행 타임라인 + 자동 트랜치 해제 |
| 건물주 | 비어 있는 상가를 미니팜으로 전환하고 싶지만 투자 비용이 부담스럽고, 운영 현황을 실시간으로 확인하고 싶다. | 공실 등록 및 프로젝트 매칭, 팜 운영 현황 모니터링 + 임대 수익 자동 정산 |
| 플랫폼 관리자 | 마일스톤 검증 결과를 한눈에 보고, 예외 상황만 개입하고 싶다. 전체 프로젝트 데이터를 집계하여 이상 징후를 조기에 포착하고 싶다. | 관리자 대시보드(마일스톤 검증 현황·이상 알림), 플랫폼 통계 대시보드 + IoT 이상 탐지 알림 |

**레이어 분리 설계:**

| 구분 | 메커니즘 레이어 (실제 구동) | 결제 입출구 레이어 (모의 처리) |
|:---|:---|:---|
| 포함 항목 | 토큰 발행·전송, 에스크로 락업·해제, AI 검증, 자동 집행, 대시보드 | 원화 입출금, KYC/AML, 증권신고서·공시, 실제 투자자 모집 |
| 구현 방식 | Polygon Amoy 테스트넷 + AI API + Vercel 공개 웹 | 화면상 '모의 결제' 안내 + mock 양식 (2027년 인가 취득 후 연동) |
| 데모 가능 여부 | 누구나 공개 URL 접속 → 실제 작동 확인 가능 | 화면 시연만 (실제 자금 이동 없음) |

### 3. 개발결과

#### 3.1. 전체시스템 흐름도

서비스 전체 흐름은 6단계로 구성됩니다:

```
[1] 투자자 → 프로젝트 선택 → 소액 청약 참여
        ↓
[2] 투자금 → 에스크로(금고)에 락업 (스마트컨트랙트)
        ↓
[3] 운영자 → 마일스톤 달성 → 증빙 자료 제출 (영수증, 현장 사진)
        ↓
[4] AI 다중 신호 검증 — 마일스톤별 필수 신호 집합 (AND 조건)
     → 전 신호 통과 시 승인 | 실패 시 재검증 1회 허용 (관리자 동시 알림)
        ↓
[5] 검증 통과 → 스마트컨트랙트가 자동으로 트랜치 자금 집행
     → 검증 2회 실패 → 자금 풀림 차단 + 관리자 수동 검토 전환
        ↓
[6] 전체 마일스톤 완료 → 운영 수익 발생 → 투자 지분 비례 배당 분배
```

##### `랜딩 페이지`
- 메인 카피: "도심 빈 공간이 스마트팜이 됩니다"
- 투자자, 건물주, 운영자 세 사용자 유형별 진입 경로 제공
- CTA 버튼: "프로젝트 둘러보기", "데모 시작"
- 하단에 플랫폼 전체 통계 및 서비스 흐름 요약 표시

##### `프로젝트 탐색 페이지`
- 지역, 모집 상태, 예상 수익률 기준 필터링
- 프로젝트 카드: 모집률, 예상 연 수익률, 마일스톤 단계, ESG 임팩트 정보 표시
- 상태 뱃지: 청약중(파랑), 운영중(초록), 완료(회색)

##### `프로젝트 상세 + 청약 페이지`
- 프로젝트 상세 정보 + 모집 현황 프로그레스바
- 소액 청약 폼: 토큰 수량 입력 → 자동 금액 계산 → 지갑 연결 → 온체인 청약
- 마일스톤 로드맵 스테퍼 (4단계)
- 에스크로 현황: 총 락업 / 해제 완료 / 잔여 (온체인 데이터)
- 최근 거래 내역 (Polygonscan Amoy 링크)

##### `라이브 대시보드`
- 4개 요약 카드: 에스크로 잔액 / 총 해제 금액 / 투자자 수 / 누적 배당
- 자금 흐름 차트 (Recharts BarChart)
- IoT 모니터링: 온도·습도·CO2·광량 실시간 표시, 이상 탐지 시 경고
- 트랜잭션 내역 + ESG 임팩트 (CO2 절감, 푸드마일 감소)
- 5초 간격 자동 리프레시

##### `데모 자동재생 페이지`
- 8단계 + 실패 케이스 3개 시연 가능
- 좌측: 스텝 진행 목록 / 우측: 실시간 상태 미러링
- [처음부터] / [다음 단계] / [자동 재생] / [실패 케이스 보기] 컨트롤

##### `관리자 페이지`
- AI 검증 현황: 마일스톤별 검증 상태, 증빙 자료 업로드, AI 검증 실행
- 재검증 알림: 재시도 발생 시 관리자 알림 + 2회 실패 시 수동 검토 대기열
- IoT 이상 알림 섹션
- 배당 실행: 매출액 입력 + 배당비율 조절 + 계산 미리보기

#### 3.5. AI 도구 활용

**시스템 구성요소로서의 AI 활용:**

| AI 활용 | 활용 단계 | 활용 방식 | 성과 |
|:---|:---|:---|:---|
| 계약서·영수증 OCR 검증 | 마일스톤 검증 단계 | 임대차 계약서·영수증 이미지 → GPT-4o/Claude Sonnet 4로 주소·면적·금액·항목·일자 JSON 추출 → 마일스톤 조건과 자동 대조 | 수작업 검토 제거, 검증 신뢰도·속도 향상 |
| 공사·수확 사진 분석 | 마일스톤 검증 단계 | 현장 사진 → Vision 모델로 설비 객체 인식(LED·센서·재배대) → 신뢰도 점수 산출 | 학습 데이터 없이 Zero-shot 적용, 다중 신호로 단독 판단 한계 보완 |
| IoT 센서 이상 탐지 | 상시 모니터링 | scikit-learn Isolation Forest + Z-score로 센서 패턴 분석 → 이상 탐지 시 마일스톤 일시 정지 | 시계열 수치에 최적, LLM 대비 빠르고 저비용 |

> **마일스톤별 신호 구조 (AND 조건):**
>
> | 마일스톤 | 비중 | 필수 신호 | 통과 조건 |
> |:---|:---|:---|:---|
> | 1. 공간 준비 | 35% | 계약서, 영수증, 사진 (영수증↔사진 교차검증) | 셋 다 통과 (AND) |
> | 2. 시운전 + 안정성 | 30% | 사진, IoT 14일 (가동률 ≥ 90%) | 둘 다 통과 (AND) |
> | 3. 첫 수확 + 판매 | 20% | 사진, 영수증, IoT | 셋 다 통과 (AND) |
> | 4. 지속 운영 | 15% | IoT 60일 (가동률 ≥ 90%), 영수증, 사진 | 셋 다 통과 (AND) |
>
> 검증 실패 시 **재검증 1회 허용** (관리자에게 동시 알림). 2회 연속 실패 시 자동 차단 → 관리자 수동 검토 전환.

### 4. 설치 및 사용 방법

**필요 소프트웨어:**
- Node.js 18+
- Python 3.8+
- Foundry (스마트컨트랙트 빌드·테스트)
- Git

**설치 및 실행:**

```bash
# 레포 클론
$ git clone https://github.com/PNU-2026-AI-Hackathon/pnuai-b-01-b301.git
$ cd pnuai-b-01-b301

# 프론트엔드 설치 및 실행
$ cd frontend
$ npm install
$ cp .env.example .env.local   # 환경 변수 설정 (DATABASE_URL, OPENAI_API_KEY, ANTHROPIC_API_KEY 등)
$ npx prisma db push           # DB 스키마 반영
$ npx prisma db seed           # 시드 데이터 생성
$ npm run dev                  # http://localhost:3000

# 스마트컨트랙트 빌드 및 테스트
$ cd ../contracts
$ forge install
$ forge build
$ forge test

# 스마트컨트랙트 배포 (Polygon Amoy)
$ forge script script/Deploy.s.sol --rpc-url $AMOY_RPC_URL --broadcast

# IoT 시뮬레이터 (선택)
$ cd ../iot-mock
$ pip install -r requirements.txt
$ python anomaly_detector.py
```

**환경 변수 (.env.local):**
```
DATABASE_URL=postgresql://...
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
NEXT_PUBLIC_AMOY_RPC=https://rpc-amoy.polygon.technology
```
